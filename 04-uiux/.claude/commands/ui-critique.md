Create a `CritiqueReport` for the current task.

Arguments: $ARGUMENTS

## Evaluate at least

- hierarchy
- consistency
- novelty / non-genericity
- accessibility
- responsiveness
- implementation cleanliness

## Required output

- numeric or labeled scores
- concrete issues
- affected layer for each issue
- prioritized next actions

## Layer mapping

- hierarchy issue -> layout
- generic style issue -> tokens/style system
- state inconsistency -> components
- hardcoded values -> token / implementation
- a11y issue -> component implementation
